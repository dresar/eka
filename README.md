## 📖 Tentang Proyek

Website ini adalah **Portfolio Profesional** resmi milik **Eka Syarif Maulana**, seorang pengembang web, desainer antarmuka (UI/UX), dan desainer grafis yang berbasis di Indonesia. Website dirancang khusus sebagai representasi digital interaktif dari keahlian, pengalaman, riwayat pendidikan, sertifikasi, serta portofolio proyek nyata yang telah diselesaikan.

### 📝 Deskripsi Singkat
Sebuah landing page portfolio responsif berkinerja tinggi yang dibangun dengan arsitektur modern berbasis Gulp, SCSS, dan Bootstrap 4. Proyek ini menyuguhkan portofolio terperinci yang mencakup desain web, periklanan, dan branding dengan transisi filter animasi yang mulus.

### 📄 Deskripsi Lengkap
Website portfolio ini bukan sekadar halaman statis biasa, melainkan pusat pameran digital yang komprehensif bagi **Eka Syarif Maulana**. Sebagai mahasiswa Teknologi Informasi di **Universitas Muhammadiyah Sumatera Utara**, Eka membangun platform ini untuk menjembatani sisi kreatif dari desain grafis (menggunakan Adobe Photoshop, Illustrator, Premiere Pro, dan Canva) dengan kemampuan teknis pemrograman (React.js, Next.js, dan optimasi database SQL).

Halaman web ini dikemas secara profesional dengan menyematkan fitur resume digital terintegrasi (dengan akses cepat cetak resume via Google Drive), sistem filter portofolio interaktif berbasis Isotope untuk menampilkan proyek Web, Periklanan, dan Branding, serta panel sertifikat dinamis dengan fitur "See More/Less" untuk meminimalkan beban pemuatan gambar (lazily loaded via JavaScript). Seluruh aset dioptimalkan secara otomatis menggunakan pipeline otomasi **Gulp** untuk kompresi gambar, minifikasi stylesheet CSS, dan kompresi file JavaScript guna menjamin performa website yang instan dan SEO-friendly.

## 🛠️ Teknologi yang Digunakan (Tech Stack)

Proyek ini memanfaatkan perpaduan teknologi tangguh untuk menjamin kecepatan, estetika, dan kemudahan pemeliharaan kode:

### Frontend Core & UI Framework
*   **HTML5 & CSS3 (Semantik & Modern)**: Fondasi struktural halaman web dengan markup SEO-friendly.
*   **Bootstrap v4.x & Affix**: Grid system responsif dan komponen navigasi pintar (sticky navbar & scrollspy) yang melekat secara elegan saat digulir.
*   **Sass / SCSS**: Mengorganisir styling secara modular menggunakan variabel, mixin, dan nesting demi kode CSS yang bersih dan rapi.
*   **jQuery & Isotope**: Menggerakkan pustaka filtering portofolio interaktif dengan efek transisi layout yang dinamis dan mulus.
*   **Themify Icons**: Pustaka ikon vektor yang ringan untuk menyempurnakan aspek visual antarmuka pengguna.

### Otomasi & Pipeline Build (Gulp Workflow)
*   **Gulp (v4.x)**: Automator tugas JavaScript yang mengelola alur kerja pengembangan dari source ke production build.
*   **Gulp Sass**: Kompilator SCSS ke CSS secara real-time.
*   **Gulp Autoprefixer**: Menambahkan vendor prefix CSS secara otomatis demi kompatibilitas lintas browser (Internet Explorer 8+, Edge, Safari, Chrome, Firefox).
*   **Gulp CleanCSS & Uglify**: Mengurangi ukuran file CSS dan JS dengan menghapus whitespace dan meminimalkan variabel untuk mempercepat waktu pemuatan.
*   **Gulp Imagemin (dengan pngquant & jpeg-recompress)**: Memproses, meresolusi ulang, dan mengompresi gambar tanpa mengurangi kualitas visual secara drastis (lossless & lossy image compression).
*   **BrowserSync**: Menyediakan server lokal pengembangan dengan fitur *Live Reload* instan di browser setiap kali ada perubahan kode.

## 🧠 AI Summaries (7 Ringkasan AI)

Berdasarkan analisis arsitektur repositori dan file kode sumber, berikut adalah **7 Ringkasan Kecerdasan Buatan (AI)** mengenai keunggulan teknik dan fungsional dari website portfolio ini:

> [!NOTE]
> ### 1. Alur Kerja Otomasi Pengembang yang Sangat Efisien
> Integrasi **Gulp 4.x** dalam proyek ini menunjukkan standardisasi alur kerja modern. Dengan satu perintah `npm start`, sistem secara otomatis mengompilasi SCSS, menyisipkan vendor prefix untuk kompatibilitas browser lama, meminimalkan aset (CSS, JS, gambar), dan meluncurkan server lokal dengan *Live Reload*. Ini mengurangi waktu pengembangan secara drastis dan memastikan efisiensi tinggi dalam siklus iterasi visual.

> [!TIP]
> ### 2. Kompresi Gambar Cerdas dan Optimal untuk Akses Cepat
> Penggunaan modul `gulp-imagemin` yang digabungkan dengan `imagemin-pngquant` dan `imagemin-jpeg-recompress` menunjukkan kepedulian tinggi terhadap optimasi performa. Gambar berukuran besar (sertifikat, portofolio, latar belakang) dikompresi secara agresif sebelum disajikan ke pengguna. Hal ini meminimalkan transfer data, mempercepat *Largest Contentful Paint (LCP)*, dan mengoptimalkan skor performa Google PageSpeed Insights.

> [!IMPORTANT]
> ### 3. Desain Antarmuka Responsif & Kompatibilitas Lintas Browser
> Dibuat di atas fondasi **Bootstrap 4** dengan konfigurasi target browser yang luas dalam `browserslist` (termasuk IE >= 8, Android >= 4, iOS >= 7, Safari >= 7). Website ini dijamin memiliki visual yang konsisten dan stabil baik diakses melalui komputer desktop lama, tablet, maupun smartphone layar kecil, memberikan pengalaman pengguna (*UX*) yang inklusif.

> [!IMPORTANT]
> ### 4. Manajemen DOM Dinamis dan Bersih via JavaScript
> Implementasi fitur sertifikat menunjukkan pendekatan cerdas untuk optimasi pemuatan halaman. Daripada memuat puluhan gambar sertifikat berukuran besar secara bersamaan ketika halaman pertama kali dibuka (yang dapat memperlambat rendering awal), JavaScript digunakan untuk memuat aset tambahan secara dinamis (*lazy rendering*) hanya ketika tombol "See More" ditekan secara manual oleh pengguna.

> [!TIP]
> ### 5. Transisi Layout Mulus dengan Isotope dan CSS Grid
> Integrasi **Isotope** untuk filtering galeri karya seni dan web memberikan sentuhan mikro-animasi yang sangat premium. Saat pengguna beralih antara tab filter (*Web*, *Branding*, *Advertising*), elemen portfolio tidak hilang mendadak melainkan bergeser dan memudar secara anggun. Ini menciptakan kesan dinamis dan hidup pada portofolio interaktif.

> [!NOTE]
> ### 6. Struktur Kode Modular dan Terorganisir
> Repositori memiliki pembagian yang sangat tegas antara aset mentah (SCSS, JS mentah, gambar resolusi tinggi) di dalam folder `assets` dan file hasil build siap-produksi di dalam folder `dist`. Pemisahan ini menjaga kerapian *codebase* dan mencegah developer dari kebingungan saat melakukan refactoring kode styling atau scripting.

> [!WARNING]
> ### 7. Kesiapan SEO & Pemasaran Personal yang Kuat
> Struktur halaman dirancang dengan menerapkan SEO On-Page yang baik. Penggunaan tag tajuk semantik (`<h1>` hingga `<h6>`), tag meta deskripsi viewport responsif, integrasi profil sosial media di header dan footer, serta pencantuman detail kontak geografis secara presisi mempermudah mesin pencari (*crawlers* Google/Bing) untuk mengindeks halaman ini dan menghubungkannya dengan pencarian bakat profesional di Indonesia.

## 🛠️ Panduan Instalasi dan Penggunaan

Ikuti langkah-langkah di bawah ini untuk memasang repositori ini di perangkat lokal Anda dan mulai menjalankan server pengembangan.

### Prasyarat Sebelum Mulai
Pastikan perangkat Anda telah terinstal:
*   [Node.js](https://nodejs.org/) (Sangat direkomendasikan versi LTS terbaru)
*   [Git](https://git-scm.com/)

### Langkah 1: Kloning Repositori
Jalankan terminal atau PowerShell Anda, lalu kloning repositori GitHub ini:
```bash
git clone https://github.com/dresar/eka.git
cd eka
```

### Langkah 2: Instalasi Dependensi
Instal semua modul Node.js yang tercantum dalam `package.json` (Gulp, BrowserSync, Sass, dll.):
```bash
npm install
```

### Langkah 3: Jalankan Server Pengembangan (Local Dev Server)
Untuk menjalankan server lokal dengan fitur pembaruan otomatis (Live Reload):
```bash
npm start
```
Perintah ini akan secara otomatis membuka peramban web (browser) Anda dan memuat situs di alamat `http://localhost:3000`. BrowserSync akan memantau setiap kali Anda mengedit file HTML, JS, atau SCSS dan memperbarui halaman secara real-time.

### Langkah 4: Membangun Proyek untuk Produksi (Production Build)
Jika Anda ingin menghasilkan aset yang dioptimalkan, dimodifikasi, dan siap diunggah ke server hosting produksi, jalankan perintah:
```bash
npx gulp build
```
Perintah ini akan membersihkan direktori `public_html/dist` sebelumnya dan mengompilasi ulang versi terkompresi dari SCSS, JS, vendor, dan gambar ke dalam folder output tersebut.

Berikut adalah visualisasi pohon direktori setelah proses reorganisasi file dipindahkan ke direktori utama (root):

```text

    │   └── vendors/           # Pustaka pihak ketiga (Bootstrap, jQuery, dll)

        ├── js/                # js minified (johndoe.min.js)
```
